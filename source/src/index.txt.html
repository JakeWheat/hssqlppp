= Overview of the library source:

The files in the folder Database/HsSqlPpp/ are all the public api files. Some of these contain the implementation code, and some are just forwarding modules.

The main interesting files are:

pretty printer: Database/HsSqlPpp/PrettyPrinter.lhs

sql types: Database/HsSqlPpp/AstInternals/TypeType.lhs

quasiquoting: Database/HsSqlPpp/SqlQuote.lhs

main sql parsing: Database/HsSqlPpp/Parsing/ParserInternal.lhs

sql lexing: Database/HsSqlPpp/Parsing/Lexer.lhs

sql ast types: Database/HsSqlPpp/AstInternals/AstInternal.ag

the code to create and query catalogs: Database/HsSqlPpp/AstInternals/Catalog/CatalogInternal.lhs

code to create and query local bindings - used for parameters and variables in plpgsql functions, as well as for passing the types of identifiers around inside statements (e.g. getting the names and types from a table ref to pass to a select list inside a select statement) Most of the logic for type checking joins is in here Database/HsSqlPpp/AstInternals/TypeChecking/LocalBindingsInternal.lhs

the type checking code is split up into multiple ag files in the following folder: Database/HsSqlPpp/AstInternals/TypeChecking/

the main hairy bit of type checking (apart from joins), includes the complicated overloaded function matching and implicit cast and conversion logic: Database/HsSqlPpp/AstInternals/TypeChecking/TypeConversion.lhs
